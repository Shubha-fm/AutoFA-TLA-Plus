# AutoFA-TLA+

AutoFA-TLA+ is a full-stack web application for creating DFA/NFA models through an automata canvas or transition table, validating the model, and generating TLA+ and TLC configuration files.

## Features

- DFA and NFA model input
- Transition-table editor
- Automata visualization
- Structural validation
- TLA+ specification generation
- TLC configuration generation
- Reachability and automata consistency checks
- Downloadable `.tla` and `.cfg` files
- Counterexample/result panel
- Single-container deployment

## Technology

- Frontend: React, Vite, Lucide React
- Backend: FastAPI, Pydantic, Uvicorn
- Deployment: Docker or Render

## Local Development

### Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

On Windows, activate the environment with:

```powershell
.venv\Scripts\activate
```

### Frontend

Open a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:5173`.

## Run with Docker

From the project root:

```bash
docker build -t autofa-tla-plus .
docker run --rm -p 8000:8000 autofa-tla-plus
```

Open `http://localhost:8000`.

## Deploy on Render from GitHub

1. Create a new GitHub repository.
2. Upload all files from this project folder.
3. In Render, choose **New +** and then **Blueprint**.
4. Connect the GitHub repository.
5. Render reads `render.yaml` and deploys the application.
6. Open the public URL generated by Render.

## API Endpoints

- `GET /api/health`
- `POST /api/validate`
- `POST /api/generate`
- `POST /api/run-tlc`

## Important Note

The current verification endpoint performs built-in structural and reachability checks so the application works without a local TLC installation. Generated `.tla` and `.cfg` files can be downloaded and executed with the official TLA+ Toolbox or command-line TLC.
